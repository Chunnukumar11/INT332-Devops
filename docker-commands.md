# Docker Commands Guide

This document includes commonly used Docker commands for managing containers, images, and system resources.

---

## Check Docker Version

Use this command to verify the installed Docker version.


docker --version


---

## Pull Docker Image

Download images from Docker Hub.


docker pull ubuntu
docker pull nginx
docker pull mysql
docker pull node


---

## List Docker Images

Display all images available in the system.


docker images


---

## Run Container

Create and run a container from an image.


docker run ubuntu


Run container interactively.


docker run -it ubuntu bash


Run container in background mode.


docker run -d nginx


---

## View Containers

Show currently running containers.


docker ps


Show all containers including stopped ones.


docker ps -a


---

## Stop Container

Stop a running container.


docker stop container_name


---

## Start Container

Start a stopped container.


docker start container_name


---

## Remove Container

Delete a container from the system.


docker rm container_id


---

## Remove Image

Delete an image from the system.


docker rmi image_id


---

## Build Docker Image

Create a Docker image using a Dockerfile.


docker build -t myimage:v1 .


---

## View Container Logs

Check logs generated by a container.


docker logs container_name


Follow logs continuously.


docker logs -f container_name


---

## Execute Command Inside Container

Access the terminal of a running container.


docker exec -it container_id /bin/bash


---

## System Information

View detailed Docker system information.


docker info


---

## Conclusion


---

## Conclusion

Docker commands allow users to manage images and containers efficiently.  
Understanding these commands is important for working with containerized applications in DevOps environments.
